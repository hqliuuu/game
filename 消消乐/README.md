# 未定主题消消乐游戏大纲

## 项目定位

本项目是在当前目录下开发的一款浏览器 2D 消消乐游戏，目录名为 `消消乐`。核心玩法参考经典三消机制：玩家交换相邻棋子，形成 3 个或以上相同元素的连线后消除、下落、补位并触发连锁得分。

美术主题以《未定事件簿》的氛围为灵感：都市律政、案件调查、证据收集、浪漫悬疑、精致角色卡面感。项目不直接复制或内置官方受版权保护素材，而是使用 AI 工具生成原创视觉资源，保持“未名市调查档案”式的主题表达。

## 推荐工具

- 开发语言：TypeScript
- 构建工具：Vite
- 游戏引擎：Phaser 3
- UI 层：HTML/CSS DOM HUD，负责菜单、弹窗、关卡结算和设置
- 素材生成：AI 图像生成工具，用于生成棋子图标、背景、特效贴图、UI 装饰
- 素材处理：Aseprite、Photoshop、Photopea 或 Krita，用于裁切、压缩、透明背景处理
- 音频工具：AI 音乐与音效生成器，或 freesound 类授权音效库，用于背景音乐、点击、交换、消除、连锁、胜负音效
- 浏览器调试：Microsoft Edge DevTools
- 版本管理：Git

## 主题素材方向

### 棋子元素

棋子建议使用 6 种基础图标，每种代表一种调查要素：

- 律师徽章：代表法律与庭审
- 证据袋：代表案件线索
- 玫瑰信笺：代表恋爱羁绊
- 放大镜：代表调查推理
- 档案夹：代表案件记录
- 晶蓝天平：代表公正与判断

### 特殊棋子

- 横向清除：闪光钢笔
- 纵向清除：庭审光束
- 爆炸清除：封存证物箱
- 同色清除：NXX 风格调查芯片

### 场景氛围

- 主菜单：夜色都市、玻璃办公楼、柔和灯光、案件档案桌面
- 游戏背景：深蓝灰法务办公室、暖色台灯、散落资料、半透明 UI 面板
- 关卡地图：未名市案件节点，以章节形式解锁

## 音乐与音效方向

### 背景音乐

背景音乐需要符合“都市律政恋爱推理”的主题，但整体听感必须舒缓，不使用紧张、压迫、恐怖或高强度追逐感音乐。音乐应服务休闲消消乐节奏，让玩家长时间游玩时不疲劳。

建议风格：

- 主菜单音乐：舒缓钢琴、轻柔弦乐、低音量电子氛围铺底，带一点夜色都市感。
- 游戏内音乐：中慢速节拍，轻柔钢琴、温暖合成器、干净的拨弦或木吉他点缀，循环自然。
- 结算音乐：胜利时使用短促明亮但不过分激烈的旋律，失败时使用温和的低落提示，不制造挫败感。

音乐约束：

- 避免战斗鼓点、急促弦乐、尖锐警报、强烈低频冲击。
- 避免过于悲伤或悬疑惊悚的旋律。
- 单曲建议控制在 60 到 120 秒，并支持无缝循环。
- 音量默认低于音效，避免遮盖消除反馈。
- 所有音乐必须为原创生成或使用明确可商用授权素材。

### 音效

音效需要清晰、轻巧、精致，突出“调查档案、玻璃、纸张、宝石、金属徽章”的质感。每个音效都应短小，不打断背景音乐。

建议音效清单：

- `ui-click`：菜单按钮点击，轻柔玻璃或纸张触感。
- `tile-select`：选中棋子，短促亮音。
- `tile-swap`：合法交换，轻微滑动和晶体碰撞。
- `tile-swap-back`：非法交换回退，柔和低音提示。
- `match-3`：普通三消，清脆宝石破碎或光点散开。
- `match-4`：生成特殊棋子，比普通三消更亮、更有层次。
- `match-5`：生成同色清除棋子，带短促上扬音阶。
- `cascade`：连锁消除，随连锁次数提高音高或叠加细小闪光音。
- `special-clear`：特殊棋子触发，使用优雅能量扫过或庭审光束质感。
- `objective-complete`：关卡目标完成，温暖确认音。
- `level-win`：胜利结算，舒展、明亮、克制。
- `level-fail`：失败结算，温和下降音，不尖锐。

音效实现建议：

- 每个音效控制在 0.1 到 2 秒。
- 同类音效准备 2 到 3 个轻微变体，避免频繁消除时重复感过强。
- 连锁音效可以根据连锁倍率做音高变化。
- 设置界面需要提供背景音乐和音效两个独立音量控制。
- 浏览器端默认等待玩家首次交互后再播放音乐，避免自动播放限制。

## 游戏规则

1. 棋盘默认使用 `8 x 8` 网格。
2. 玩家每次只能交换上下左右相邻的两个棋子。
3. 交换后必须形成至少一组 3 连，否则棋子回退。
4. 3 个相同棋子连成横线或竖线时消除。
5. 4 连生成一个方向清除特殊棋子。
6. 5 连生成一个同色清除特殊棋子。
7. T 型或 L 型匹配生成爆炸特殊棋子。
8. 消除后上方棋子下落，空位由新棋子补充。
9. 下落后如果继续形成匹配，则自动连锁消除。
10. 每关有步数限制、目标分数或指定收集目标。

## 胜负条件

### 胜利

满足当前关卡目标即胜利，例如：

- 达到指定分数
- 收集指定数量的证据袋
- 清除所有封锁档案格
- 在限定步数内触发指定次数连锁

### 失败

- 步数耗尽且目标未完成
- 限时关卡中倒计时结束

## 游戏说明

玩家将作为新人律师参与案件调查，通过完成消消乐关卡推进章节。每个关卡代表一次调查行动，例如整理证据、还原线索、准备庭审、解锁角色剧情片段。

游戏节奏以轻量休闲为主，每局目标时长控制在 2 到 4 分钟。核心体验包括：

- 精致主题棋子带来的收集感
- 连锁消除带来的爽感
- 章节关卡带来的推进感
- 案件目标和恋爱羁绊带来的主题代入感

## 开发组件划分

### 核心游戏组件

- `BoardManager`：创建棋盘、维护棋子矩阵、检测可交换状态
- `Tile`：单个棋子对象，包含类型、坐标、选中状态和动画状态
- `MatchResolver`：检测 3 连、4 连、5 连、T 型和 L 型匹配
- `CascadeSystem`：处理消除、下落、补位和连锁
- `SpecialTileSystem`：创建和触发特殊棋子效果
- `ScoreSystem`：计算基础分、连锁倍率和目标进度
- `LevelManager`：读取关卡配置、判断胜负、进入下一关

### 表现组件

- `AnimationController`：交换、消除、下落、爆炸、闪光等动画
- `EffectLayer`：粒子、光效、连锁提示、特殊技能表现
- `AudioManager`：背景音乐、点击音效、交换音效、消除音效、连锁音效、胜负音效、音量设置
- `AssetLoader`：统一加载图片、音频、字体和配置

### UI 组件

- `MainMenu`：开始游戏、继续游戏、设置入口
- `HUD`：分数、步数、关卡目标、暂停按钮
- `LevelSelect`：章节与关卡选择
- `ResultModal`：胜利、失败、星级、奖励展示
- `SettingsPanel`：音量、画质、重开、返回菜单

### 数据组件

- `levels.json`：关卡目标、步数、棋子类型池、障碍物配置
- `assets.manifest.json`：图片、音频、字体等资源清单
- `save.json` 或 `localStorage`：玩家进度、本地设置、最高分

## AI 素材生成规范

为避免直接复刻官方素材，AI 生成提示词应强调原创设计：

- 使用“律政恋爱推理手游氛围”“都市悬疑”“精致卡牌式 UI”“调查档案主题”等描述
- 不要求生成官方角色原图、官方 logo 或高度相似的卡面
- 棋子图标保持原创符号设计，统一光泽、边框、尺寸和透明背景
- 背景图保留足够暗部和留白，避免干扰棋盘可读性
- 所有导出图片统一命名、统一分辨率、统一压缩格式

## 开发里程碑

1. 完成 README 和玩法规格。
2. 搭建 Vite + Phaser 项目骨架。
3. 实现基础棋盘、交换、匹配、消除、下落和补位。
4. 接入分数、步数、胜负判定和关卡配置。
5. 生成并接入第一批主题棋子、背景和音效。
6. 加入特殊棋子、连锁动画和粒子效果。
7. 完成主菜单、关卡选择、HUD、结算界面。
8. 使用 Edge 浏览器进行本地试玩和视觉 QA。

## 验收标准

- 可以在浏览器中运行。
- 棋盘不会在开局时自动出现已匹配组合。
- 每次合法交换都能正确消除并补位。
- 非法交换会回退。
- 连锁、特殊棋子、分数和步数表现正确。
- UI 在桌面与常见移动宽度下不重叠、不遮挡棋盘。
- 主题素材风格统一，棋子辨识度清晰。
